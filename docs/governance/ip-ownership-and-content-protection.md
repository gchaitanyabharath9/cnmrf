# IP Ownership & Content Protection

## 1. Overview
CNMRF is an open-source, platform-neutral architectural reference framework. While the code and patterns are open, the intellectual property of the compilation, structure, and original documentation is defined here to ensure clarity on usage and attribution.

## 2. Protected Content
The following assets are the intellectual property of the author, licensed under Apache 2.0:
*   **Architectural Compilations:** The specific organization of patterns, diagrams, and governance models.
*   **Diagrams:** All Mermaid, Visio, and UML source files.
*   **Written Text:** All documentation, guides, and strategic narratives.

## 3. Usage & Attribution
When utilizing CNMRF assets in other works, you must:
1.  **Retain Copyright Notices:** Keep the Apache 2.0 license header in code files.
2.  **Cite the Source:** Reference "CNMRF by Chaitanya Bharath Gopu" in documentation.
    *   *Example:* "Based on the Cloud-Native Modernization Reference Framework (v1.1.1)."

## 4. Private & Excluded Content
The following are NOT included in this open-source release and must remain private if created locally:
*   **Secrets/Credentials:** `.env` files, actual certificates, or private keys.
*   **Customer Data:** Any specific implementation details related to proprietary business logic or user data.

## 5. Release Provenance
Official releases are marked by:
1.  **Signed Git Tags:** `vX.Y.Z` signed by the author.
2.  **GitHub Releases:** Archived source code bundles.
3.  **Freeze Declarations:** Documentation snapshots (e.g., `docs/releases/v1.1.1-freeze.md`).
