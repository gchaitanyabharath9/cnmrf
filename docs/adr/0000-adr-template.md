## Status Lifecycle

- **Proposed:** ADR is drafted and under review
- **Accepted:** ADR is approved and should be followed
- **Deprecated:** ADR is no longer recommended but not yet replaced
- **Superseded:** ADR has been replaced by a newer ADR (reference provided)

## Best Practices

- **Be concise:** ADRs should be readable in 5-10 minutes
- **Be specific:** Avoid vague language; state decisions clearly
- **Document rationale:** The "why" is more important than the "what"
- **Consider alternatives:** Show that options were evaluated
- **Update status:** Keep ADRs current as decisions evolve
- **Link liberally:** Reference related ADRs and documentation
