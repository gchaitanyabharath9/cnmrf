# CNMRF v1.1.0 Frozen Reference Baseline

**Date:** 2026-01-06
**Status:** FROZEN (v1.1.0)

## 1. Declaration
The Cloud-Native Modernization Reference Framework (CNMRF) is hereby declared feature-complete and **frozen at Version 1.1.0**. This release incorporates the Networking & Traffic Management extension.

CNMRF is an **open-source, platform-neutral architectural reference framework**. It is NOT a product, SaaS, or commercial offering.

## 2. Included Scope (v1.1.0 Baseline)
Includes all v1.0 scope, plus:
*   **Networking:** DNS Strategies (Split-Horizon, GTM), Load Balancing (L4/L7), Ingress Patterns.
*   **Traffic Management:** Routing (Host/Path/Header), API Versioning, Route-as-Code Governance.
*   **Migration Safety:** Traffic Cutover Playbooks (DNS vs Gateway), Rollback Strategies.
*   **Governance:** DNS/Routing Change Control and Evidence Requirements.

## 3. Post-Freeze Exclusions
The following areas remain OUT OF SCOPE:
*   SaaS billing logic.
*   Vendor-specific implementation details.
*   UI/Frontend component libraries.

## 4. Versioning Policy
*   **v1.1.x:** Maintenance only. No new scope.
*   **v2.0.0:** Major architectural changes.
